Responsible disclosure.

Found a vulnerability in this app, the deployed service, or the model backend it talks to? Report it through the channel below and we'll respond within five working days. We don't run a paid bounty programme, but we credit reporters in the changelog where they consent.

How to report

Email the maintainer with the words "[security]" in the subject line. Include enough detail to reproduce — affected URL, payload, impact, and any logs. PGP key on request.

muhammadabdullahmughal740@gmail.com

Response timeline

Acknowledgement within 72 hours. Triage within 5 working days. Patch ETA depends on severity — critical issues get same-week fixes, lower-severity ones land in the next release window. We'll keep you in the loop on progress.

In scope

  • The Dark Prompt web app (dark-prompt.onrender.com).
  • Authentication / session / 2FA bypasses.
  • SQL injection, RCE, SSRF, deserialisation, CSRF in any view.
  • Privilege escalation across user boundaries.
  • Information leakage from /api/chat/ or the activity log.
  • Stored / reflected XSS in any rendered template.

Out of scope

  • The model's output (we know — that's what it generates by design).
  • DoS / brute-force at scale (the rate-limiter is intentional).
  • Issues requiring physical access to the user's machine.
  • Self-XSS and clickjacking on pages with no sensitive actions.
  • Vulnerabilities in third-party dependencies that don't have a known exploit path through this app.

Safe-harbour pledge

// Good-faith research within this scope will not be pursued legally.
// Don't pivot from a finding into other accounts, data, or services.
// Don't hold a vulnerability back in exchange for payment — coordinated disclosure or full disclosure are both fine.
// Don't publish details before a patch ships unless we miss the timeline above by more than 30 days.
// We'll credit you in the changelog if you'd like.
