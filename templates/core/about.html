Defenders read the same code attackers write.

Dark Prompt is a local-first frontend for an offline LLM that generates the kind of source code AV / EDR vendors, detection engineers, and CTF players already need to study. Nothing leaves your network. Nothing is uploaded. The model lives on your machine, and so does every artefact it produces.

Why this exists

Static malware samples age fast. Live coverage requires fresh variants — the same families seen in the wild, but expressed differently enough to expose gaps in signatures, heuristics, and behavioural rules.

Generating those variants by hand is slow and inconsistent. A scoped LLM generates them on demand, with knobs for technique, language, and target OS. The output goes straight into a sandbox, a YARA rule, a Sigma stub — wherever the team's pipeline actually consumes it.

Who it is for

  • AV / EDR researchers probing detection coverage.
  • Authorised red teams needing scoped tooling for a target they own or have written permission to assess.
  • CTF players + students studying offensive techniques in a controlled environment.
  • Detection engineers building YARA, Sigma, and EDR signatures against fresh artefacts.

Who it is not for

  • Targeting third parties without explicit, documented authorisation.
  • Bypassing endpoint protection on systems you do not own.
  • Mass distribution of generated payloads.
  • Anything illegal under the laws of the operator's jurisdiction.

The Acceptable Use Policy is binding. Subscription tiers gate access to higher-impact capabilities behind a separate ethical agreement.

How it works

  1. 01You write a prompt describing the artefact — language, target, technique, constraints.
  2. 02The request hits a POST /generate endpoint on a local LLM bound to 127.0.0.1.
  3. 03The model returns source code; the console highlights it, lets you copy / download / hex-dump.
  4. 04Conversations are stored locally; nothing is sent to a third-party API.
