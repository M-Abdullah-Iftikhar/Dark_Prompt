    // THESIS
    
Defenders read the same code attackers write.

    

      Static malware samples age fast. Live AV / EDR coverage depends on a
      constant stream of fresh variants — the same families seen in
      the wild, expressed differently enough to expose gaps in signatures,
      heuristics, and behavioural rules. Generating those by hand is slow
      and inconsistent. Dark Prompt is a local-first frontend for an offline
      LLM that produces them on demand, scoped by language, technique, and
      target OS.
    

  


  

    

      // FOR
      
Defensive researchers

      
AV / EDR detection engineers, authorised red teams, CTF players,
         academic security courses.

    

    

      // AGAINST
      
Production deployment

      
Targeting third-party infrastructure without authorisation, mass
         distribution, anything outlawed in your jurisdiction.

    

    

      // LOCAL
      
Nothing leaves the box

      
Model + artefacts stay on your machine. Conversations live in your
         own database. No third-party telemetry, no cloud inference.

    // FAQ
    
Frequently asked.

    
If something here doesn't answer your question, the
       contact page has more detail.

  


  

    

      
        Is this legal?
        ›
      
      

        
Generating source code that demonstrates malware techniques is a
           well-established part of defensive security research — it's how
           AV vendors, detection engineers, and academic courses build
           coverage. What's not legal is using that code to attack
           systems you don't own or have written permission to test. Dark
           Prompt is gated behind an
           Acceptable Use Policy
           that every user signs at registration, and higher tiers add a
           per-subscription ethical-use form.

      

    


    

      
        Where does my data go?
        ›
      
      

        
Conversations live in this app's database — they're not sent to
           OpenAI, Anthropic, or any third-party inference provider. The
           generation model itself runs on infrastructure you control (a
           Kaggle notebook, an HF Space, or your own GPU box) and is reached
           via a Gradio HTTPS tunnel. The only third-party calls Dark Prompt
           makes are: Brevo for transactional email, and Groq for
           generating short chat titles. Both are off by default if their
           API keys are unset.

      

    


    

      
        What models does it run?
        ›
      
      

        
Two backends today: an Assembly generator (TASM / MASM / NASM
           dialects, x86 + x86_64 + 16-bit DOS) and a C generator (targeting
           x86_64-w64-mingw32-gcc for Windows PE output). Each
           conversation is locked to one language at first send. Add more
           backends by pointing extra LLM_API_URL_* env vars at
           your own Gradio spaces — see the
           status page for the live
           endpoints.

      

    


    

      
        Can I plug in my own model?
        ›
      
      

        
Yes. The chat backend speaks the standard
           gradio_client protocol with a
           (instruction, max_tokens, temperature) signature.
           Any HuggingFace Space, Kaggle notebook, or local Gradio app
           exposing that interface drops in. Set
           LLM_API_URL in your environment, restart, and the
           Console talks to it.

      

    


    

      
        Does it actually compile or run the code?
        ›
      
      

        
Compile, optionally — yes, via local toolchains (nasm,
           gcc, jwasm, pwsh, etc.) when
           they're on the host's $PATH. The Console's
           Analyse button does static analysis (syntax check + a
           suspicious-API scan with MITRE tags); Compile turns the
           source into a binary in a per-build temp directory you can
           download. Execution / detonation is intentionally not
           shipped in the box — that's the job of a real sandbox like
           Cuckoo or a third-party API like Hybrid Analysis.

      

    


    

      
        Why subscription tiers if it's a research tool?
        ›
      
      

        
The tiers are an ethical-use gate, not a paywall. Sniffer is free
           and gives you the full Console; Exploit and Zero Day add a
           per-subscription ethical-use form (organisation, role, signed
           acknowledgements) that scopes the tool to a specific operator
           rather than a single email address. Higher token limits come
           along with that.

      

    


    

      
        What's it built with?
        ›
      
      

        
Django 5.x on Python 3.12, Postgres for production state, SQLite
           locally. Static files via WhiteNoise. Email through Brevo's HTTPS
           API. Auth uses Django's built-ins plus a custom email-or-username
           backend, TOTP-based 2FA (RFC 6238, no third-party 2FA library),
           cache-backed rate limiting, and personal API keys for /api/chat/.
           Frontend is vanilla JS — no build step, no framework. Deployed on
           Render free tier, autodeploy from main via GitHub
           Actions.
